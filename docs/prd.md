# CLI 产品需求文档 (PRD) V 1.0
`web2md` CLI v1.0 (MVP)
## 1. 产品定位与目标
**web2md** 是一个零依赖、跨平台的命令行工具。其核心目标是将高质量的网页文章转化为完全离线化的 Markdown 笔记，并自动抓取相关的图片和视频资源，完美适配 Obsidian、Typora 等本地知识管理软件。

## 2. 用户故事 (User Stories)
- **作为核心用户（知识库重度玩家）**，我希望通过一行命令就能将网页连同图片/视频下载到本地特定的文件夹结构中，以便于我在 Obsidian 中构建永久的、无死链的个人知识库。
    
- **作为批量收集者**，我希望工具在下载遇到个别失效图片时能够宽容处理并继续运行，以便于我不需要因为单一资源的报错而前功尽弃。
    
- **作为 CLI 工具使用者**，我希望在下载大量媒体资源时能看到清晰的进度条和状态反馈，以便于我知道程序仍在正常工作且没有卡死。
    

## 3. 核心功能说明 (Core Features)
### 3.1 命令行交互设计
工具遵循标准 POSIX 规范，提供详尽的 `--help` 提示。
- **基础语法**：`web2md <URL> -n <文档名称>`
- **参数定义**：
    - `<URL>` (必填)：需要抓取的目标网页绝对路径。
    - `-n, --name` (必填)：生成的 Markdown 文件名（不含 `.md` 后缀）。若未提供，程序打印友好错误并退出。
    - `--strict` (选填)：严格模式开关。默认关闭（即宽容模式）。开启后，任何资源下载失败将直接触发错误并中断程序。   

### 3.2 目录与文件生成规则
- **存储位置**：在执行命令的当前目录下生成。
- **结构约束**：
    - 生成主文件：`./<name>.md`
    - 生成资源夹：`./assets/` （若无则自动创建）。
        
- **媒体重命名规则**：从网页解析到的媒体资源存入 `assets` 时，若发现同名文件（如 `image.png`），自动重命名为 `image_1.png`, `image_2.png`，保留原格式后缀以确保编辑器解析正常。

### 3.3 内容提取与转换逻辑
- **正文提取**：采用 Readability 算法清洗网页，剔除导航栏、侧边栏、页脚及广告等非正文 DOM 元素。
- **资源识别**：
    - 图片：抓取正文内的 `<img src="...">` 标签。
    - 视频：仅抓取带有明确后辍的直链视频（如 `<video src="*.mp4">` 或 `<source src="*.webm">`），忽略流媒体（如 m3u8）或无直链的 iframe 播放器。
        
- **路径重写**：在转换为 Markdown 格式时，将 HTML 中的媒体绝对路径全部替换为本地相对路径（如 `![img](./assets/image_1.png)`）。 

### 3.4 元数据展示
生成的 Markdown 文件需在标题下方记录来源信息，便于后期追溯，同时避免 YAML Frontmatter 在普通阅读器中显得突兀：

```Markdown
# <文章标题>

> 原文链接：<目标网页的 URL>
> 抓取时间：<执行抓取动作的本地时间，格式 YYYY-MM-DD HH:mm:ss>
```

## 4. 异常处理与体验设计 (Edge Cases & UX)
- **场景 A：网络无法访问或 404**
    - **动作**：程序立即安全退出。
    - **终端提示**：`❌ 错误：无法访问该网页，请检查网络连接或链接是否正确。`
        
- **场景 B：遭遇强反爬或重度动态页面（无法提取有效正文）**
    - **动作**：提取失败，生成仅包含标题/来源信息和原链接的“空” Markdown 文件。
    - **终端提示**：`⚠️ 警告：连接成功但未提取到有效正文。该网站可能存在反爬机制或为动态渲染页面，已保存原始链接。`
        
- **场景 C：宽容模式下部分资源下载失败**
    - **动作**：跳过失败的资源，Markdown 内的相应位置保留原始网络 URL 或替换为 `[资源下载失败]` 的文本提示，程序继续执行至完毕。
    - **终端提示**：结束时输出 `⚠️ 警告：文章已保存，但有 X 个资源下载失败，已在文中保留原链接。`
- **UI 反馈**：资源下载阶段，终端需展示并发进度条（如 `[2/15] 正在下载: cover.jpg...`）。
    

## 5. 核心验收标准 (Acceptance Criteria)
1. **纯净度验收**：使用 Typora/Obsidian 打开生成的 `.md` 文件，核心正文阅读连贯，无明显网页杂质代码。
2. **离线验收**：断开本机网络，打开 `.md` 文件，所提取的图片与视频均能正常显示或播放。
3. **防呆验收**：不输入 `-n` 参数时，程序不会崩溃，而是优雅地弹出完整的 Help 帮助文档。
    

## 6. 技术实现基准 (Technical Baseline)
- **开发语言**：Go (Golang) —— 确保最终产物为单文件可执行程序，零环境依赖。
- **建议技术栈**：`spf13/cobra` (CLI骨架), `go-readability` (正文清洗), `goquery` (DOM 操作与资源重写), `html-to-markdown` (格式转换), `progressbar` (终端UI)。
